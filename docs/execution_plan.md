# Workflow
This section explains the workflow and the tools used in each domain

## Account Management
1. Once launch the web app, the user will need to register a user account using email/username and password.
2. The recommended login tool and framework used here is Clerk, with only email login is enabled.

## Company Profile Generation
1. The once the user logins, he needs to answer a designed short questionaire that includes questions like company revenue and target grant amount and upload some essential documents such as SSM and financial statements in the Company Profile tab.
2. Our first Gemini-powered agent, the "Extractor Agent" will read through the information and generate a JSON formatted company profile that signifies the company background.
3. The Company Profile is then successfully generated.

## Grant Scrapping
1. The second Gemini-powered agent, namely the "Scout Agent" will scrap through the Internet in our backend to search for grant-relevant information.
2. The Scout Agent will operate 8-hours daily, using scheduled backend jobs and Gemini extraction when live LLM extraction is enabled.
3. Throughout the work each day, the Scout Agent will spend 3 hours finding informations in some biggest grant offering website like MDEC, smifunding.com and smekom.my in Malaysia and both national and provincial SME service platform for China.
4. The Scout Agent is expected to extract essential information like grant requirements, description, nationality, offering amount etc, and parse these information into a formatted JSON object and store into the grant database.
5. Then the Scout Agent will update the grant database daily and lively, to ensure no outdated or overdue grant application is staying in the grant database.
6. Most of the information in the websites and on the Internet are unstructured data, the Scout Agent is capable of handling structured data into formatted JSON objects.

## Grant Displaying
1. In the "Grant" tab, the third Gemini-powered agent, which is the "Evaluator Agent" will surf through the grant database to look for several grants that suits the company based on the company profile.
2. Then the Evaluator Agent will rank the grants based on a suitability score.
3. The ranked grant list will be displayed to the user.

## Grant Application
1. In the "Grant" tab, if the user is interested in a grant, he can click on the grant item, and the system will lead the user to another page endpoint that shows the details and information about the grant, and the application procedures.
2. The procedures include the documents needed to be prepared by the company.
3. The documents could be categorized into 2, namely "Generated" or "Attached". Generated documents are like proposal or pitch deck which could either be uploaded and prepared by the company or generated by our last Gemini-powered agent, the "Drafter Agent"; whereas the Attached documents are documents that can't be generated by the Drafter Agent such as the company SSM or financial statements.
4. Note that if the company SSM or financial statements or other essential documents are already uploaded while registering company profile, they will automatically be retrieved from the document database and linked to the grant application page.
5. The user can always download all/any of the documents from the grant application endpoint anytime.
6. The user needs to manually submit all documents to an attached grant website.
